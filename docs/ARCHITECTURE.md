# 阿里云Serverless CI/CD 架构设计

## 1. 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                        用户层                                │
│  GitHub代码仓库 → GitHub Actions → 阿里云Serverless         │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   静态网站    │    │   SPA应用    │    │   API服务    │
│   OSS+CDN    │    │   OSS+CDN    │    │  函数计算FC  │
└──────────────┘    └──────────────┘    └──────────────┘
```

## 2. 核心组件

### 2.1 GitHub Actions
- **触发条件**: push到main分支、手动触发
- **核心任务**: 
  - 检测变更项目
  - 自动识别项目类型
  - 构建和部署

### 2.2 阿里云OSS
- **用途**: 静态网站托管
- **特性**: 
  - 自定义域名
  - CDN加速
  - 自动HTTPS

### 2.3 阿里云函数计算
- **用途**: Node.js API托管
- **特性**:
  - 按需付费
  - 自动扩缩容
  - 多可用区部署

## 3. 项目类型检测逻辑

```
projects/
├── my-site/          # 有index.html → 静态网站
├── my-react-app/     # 有package.json + react → SPA
└── my-api/           # 有package.json + express → API
```

## 4. 部署流程

```
1. 开发者推送代码
2. GitHub Actions触发
3. 检测变更的项目
4. 识别项目类型
5. 执行对应构建脚本
6. 部署到阿里云
7. 输出访问地址
```

## 5. 成本优化

| 优化策略 | 说明 |
|---------|------|
| 按量付费 | 函数计算仅按调用次数计费 |
| CDN缓存 | 静态资源缓存，减少OSS流量 |
| 构建缓存 | GitHub Actions缓存node_modules |

## 6. 安全设计

- **RAM子账号**: 最小权限原则
- **Secrets管理**: GitHub Secrets存储密钥
- **HTTPS强制**: CDN配置HTTPS跳转
